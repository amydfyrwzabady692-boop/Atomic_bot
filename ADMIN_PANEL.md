# راهنمای پنل مدیریت Atomic Bot

پس از استقرار، مدیر اصلی در تلگرام دستور `/admin` را می‌زند. امکانات پنل:

1. مدیریت بسته‌های جم: افزودن، تغییر عنوان/قیمت/موجودی و فعال یا غیرفعال‌کردن
2. مدیریت پک سنس PC و Mobile: افزودن، تغییر عنوان/قیمت و وضعیت
3. مدیریت محصولات و دسته‌بندی‌های فروشگاه
4. ساخت و حذف کد هدیه
5. ساخت و حذف کد تخفیف
6. چت و پاسخ به تیکت‌های پشتیبانی
7. افزودن و حذف دپارتمان و تنظیم آیدی پشتیبانی
8. مشاهده و تایید/رد رسیدهای کارت‌به‌کارت سفارش و کیف پول
9. جستجوی کاربر و سفارش
10. فهرست کاربران دارای موجودی، زیرمجموعه و شماره کارت تاییدشده
11. شارژ همگانی کیف پول کاربران
12. ارسال پیام همگانی
13. تنظیم زرین‌پال، callback و فعال/غیرفعال‌کردن درگاه
14. تنظیم کارت‌به‌کارت، شماره کارت، صاحب کارت و بانک
15. ویرایش نام فروشگاه، متن خوش‌آمد و متن پشتیبانی
16. افزودن و حذف مدیران تلگرامی ربات
17. آمار کل کاربران، خریداران، موجودی، تعداد فروش و جمع فروش

مدیر اصلی از `ADMIN_CHAT_ID` در فایل `.env` خوانده می‌شود و قابل حذف از پنل
نیست. مدیران بعدی از بخش «تنظیمات ← مدیران ربات» با شناسه عددی تلگرام
افزوده می‌شوند. داشتن Telegram Premium شرط فنی تلگرام برای مدیرشدن نیست؛
هر حسابی که شناسه عددی آن ثبت شود دسترسی مدیر می‌گیرد.

## استقرار روی VPS

در پوشه پروژه روی سرور:

```bash
git pull origin main
docker compose up -d --build
docker compose logs --tail=100 bot
curl -fsS https://botatomic.atomicshop.ir/health
```

مهاجرت‌های دیتابیس در شروع ربات به‌شکل idempotent اجرا می‌شوند و اطلاعات
فعلی PostgreSQL را حذف نمی‌کنند. تنظیمات جدیدی که از پنل تغییر می‌کنند داخل
جدول `BotSettings` ذخیره می‌شوند و بعد از restart باقی می‌مانند.

برای زرین‌پال، مقدار callback باید یک آدرس HTTPS عمومی باشد؛ مقدار پیشنهادی:

```text
https://botatomic.atomicshop.ir
```

ربات مسیر کامل callback هر سفارش را خودش می‌سازد. لینک پرداخت هم به‌صورت
متن قابل کپی و هم دکمه مستقیم «باز کردن درگاه پرداخت» نمایش داده می‌شود.
